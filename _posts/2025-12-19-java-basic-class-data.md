# 클래스와 데이터

## 클래스 도입의 필요성

- 김영한님의 실전 자바 강의 중 클래스와 데이터 챕터를 학습하며 클래스가 왜 필요한지, 그리고 메모리상에서 어떻게 동작하는지를 정리함

### 변수와 배열 사용의 한계
* 변수 사용
  * 학생이 늘어날 때마다 변수를 계속 추가해야 하고 출력 코드도 매번 수정해야 함
* 배열 사용
  * 데이터 관리가 수월해지지만 구조적인 문제가 발생함
  * 이름, 나이, 성적이 서로 다른 배열에 쪼개져 있어 인덱스 관리가 어려움
  * 데이터 변경 시 인덱스 순서를 정확하게 맞춰야 하며 실수할 위험이 큼

```java
// 배열 사용의 한계: 데이터가 쪼개져 있어 관리가 위험함
String[] studentNames = {"학생1", "학생2"};
int[] studentAges = {15, 16};
int[] studentGrades = {90, 80};
```

### 클래스의 등장


* 학생이라는 개념을 하나로 묶어 관리할 필요성이 대두됨
* `Student` 클래스를 만들어 이름, 나이, 성적을 한 곳에서 관리함

  ```java
  public class Student {
      String name;
      int age;
      int grade;
  }
  ```

## 용어 정리

### 클래스 (Class)

* 정의
  * 객체를 생성하기 위한 설계도 또는 틀
  * 객체가 가져야 할 속성(멤버 변수)과 기능을 정의함


* 멤버 변수 (Member Variable)
  * 클래스 내부에 정의된 변수
  * 데이터베이스의 필드와 유사한 개념


### 객체 (Object)와 인스턴스 (Instance)

* 객체
  * 클래스에서 정의한 속성과 기능을 가진 실체


* 인스턴스
  * 특정 클래스로부터 생성된 객체임을 강조할 때 사용
  * 예를 들어 `student1`은 `Student`의 인스턴스라고 표현함

<br/><br/>

## 객체 생성과 메모리 구조

### 객체 생성 과정

* new 키워드와 참조값
  * `new Student()`를 호출하면 메모리에 실제 객체가 생성되고 참조값(주소)이 반환됨
  * 변수에는 객체 자체가 아니라 이 참조값이 저장됨



  ```java
  // x001 참조값 반환 및 저장
  Student student1 = new Student();
  // 참조값을 통해 접근
  student1.name = "학생1";
  ```

### 참조값의 이해

* 대입의 원칙
  * 자바는 항상 변수의 값을 복사해서 대입함
  * 객체를 대입하는 경우 객체 자체가 복사되는 것이 아니라 참조값(주소)만 복사됨


* 객체 접근
  * 점(.) 연산자를 사용함
  * 변수에 저장된 참조값을 통해 실제 메모리에 존재하는 객체에 접근함

<br/><br/>

## 객체 배열과 리팩토링

### 배열과 객체의 결합

* 구조
  * 배열도 객체를 직접 저장하는 것이 아니라 객체의 참조값을 저장함
  * 배열 생성 시에는 배열의 크기만큼 참조값을 담을 공간이 확보됨



  ```java
  Student[] students = new Student[2];
  // student1의 참조값(x001)이 복사되어 들어감
  students[0] = student1;
  ```

### 리팩토링 효과

* 향상된 for문 활용
  * 배열을 사용하면 반복문을 통해 로직을 깔끔하게 관리할 수 있음
  * 객체 지향적인 코드로 데이터와 기능 관리가 수월해짐



  ```java
  // 배열 도입 후 최적화된 출력 로직
  for (Student s : students) {
      System.out.println("이름:" + s.name + " 나이:" + s.age + " 성적:" + s.grade);
  }
  ```

## 요약 정리

### 데이터 관리의 진화

* 발전 과정
  * 변수 → 배열 → 클래스로 발전함
  * 클래스를 통해 관련된 데이터를 하나로 묶어 관리할 수 있게 됨
  * 사람이 이해하기 쉬운 구조로 프로그래밍이 가능해짐

### 자바 메모리 대원칙

* 참조값 복사
  * 자바에서 변수의 대입은 항상 값을 복사해서 전달함
  * 기본형(Primitive)은 실제 값을 복사함
  * 참조형(Reference)은 객체의 위치를 가리키는 참조값(주소)을 복사함


* 객체와 참조
  * 변수에는 객체가 들어있는 것이 아님
  * 변수에는 객체에 접근할 수 있는 리모컨(참조값)이 들어있음
  * 이 참조값을 통해 메모리에 있는 실제 객체에 접근하고 데이터를 조작함
