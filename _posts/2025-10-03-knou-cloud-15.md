# 강의록

---

### Q1

- 다음은 무엇에 대한 설명인가?
    - 안정적인 서비스를 위해 서비스 대상과 리스크를 분석하고 컴**퓨팅 리소스를 적절하게 배치 및 설계**하는 과정

    a. 클라우드 아키텍처링

### Q2

- 다음 중 클라우드 서비스의 **효율성**과 **가용성**을 향상 시키는 아키텍처라고 할 수 없는 것은?
    
    a. 가상 머신

    - 클라우드 서비스의 효율성과 가용성을 향상 시키는 아키텍처

### Q3

- ML 수행 과정이 올바른 순서로 나열된 것은?

    a. 데이터 수집 → 데이터 전처리 → 데이터 학습 → 모델 평가

### Q4

- 다음 중 **클라우드 버스팅**에 대한 설명으로 올바르지 않은 것은?

    a. 버스트 아웃을 통해 퍼블릭 클라우드의 데이터를 복제하여 프라이빗 클라우드로 전송한다
    - 클라우드 버스팅
        - 프라이빗 클라우드에 구성된 서비스 구조와 동일한 구성을 퍼블릭 클라우드에 구성함
        - 클라우드 버스팅을 실행해야 할 임계 값 설정 및, 대상 데이터와 애플리케이션 설정함
        - 버스트 인을 통해 데이터 동기화를 진행함

### Q5

- 엣지 계층에 NaaS(Network as a Service)와 NSaaS(Network Security as a Service)를 결합하여 제공하는 기술은?

    a. SASE

### Q6

- '부하 분산 장치 만들기'에서 IP 주소 및 포트 조합으로 전송되는 수신 트래픽을 백엔드 풀 인스턴스 그룹에 분산하려고 할 때 사용하는 것은?

    a. 부하 분산 규칙

### Q7

- 다음은 Blob Storage에서 파일을 삭제하는 파이썬 코드이다. **연결 문자열**이 위치하는 곳은?
    
    ![image.png](/assets/img/knou/cloud/2025-10-03-knou-cloud-15/image.png)
    
    a. connection string

### Q8

- 부하 분산 장치 만들기 과정에서 설정하는 상태 **프로브**의 역할은?
    
    ![image.png](/assets/img/knou/cloud/2025-10-03-knou-cloud-15/imagea.png)
    
    a. 리소스 상태 체크

### Q9

- 다음 어떤 스토리지 단위에 대한 설명인가?
    - 데이터가 **객체 단위로 구조화**되는 스토리지로 데이터와 메타데이터로 구성된다.
    - 각각의 데이터 객체는 **고유 ID**로 구분된다.

    a. 오브젝트

### Q10

- 다음의 절차는 무엇에 대한 설명인가?
    - 기존의 VM을 복제하여 새로운 물리 서버에 생성한 VM으로 이관하여 **다운타임을 최소화**하는 작업

    a. 무중단 서비스 재배치

### Q11

- **고속의 데이터 접근 서비스**를 제공하기 위해 저장 장치와 서버를 **파이버 채널**을 사용하여 연결한 네트워크는?

    a. SAN

### Q12

- 다음 중 **리소스 풀링**에 대한 설명으로 올바르지 않은 것은?

    a. 리소스 풀의 리소스의 물리적 위치를 공개하여 인접한 리소스를 사용자가 선택한다

    - 리소스 풀링
        - 가용한 리소스를 담아두는 공간을 사용함
        - 도시, 국가 단위의 계층적 구조를 가짐
        - 사용자는 리소스를 빌리고 사용한 만큼 비용을 지불함

### Q13

- Azure ML Studio에서 데이터의 **데이터 형을 변경**하기 위해 사용하는 블록은?

    a. Edit Metadata

### Q14

- 다음 중 **로드 밸런싱**이 요구되는 상황으로 올바르지 않은 것은?

    a. 이용자의 증가량이 확정적으로 예측되는 상황

    - 로드 밸런싱이 요구되는 상황은
        - 갑작스럽게 이용자가 폭증하는 상황
        - 서비스의 확장 및 축소가 필요한 상황
        - 하나의 서버에 서비스 요청이 집중되는 상황

### Q15

- Blob Storage를 사용하여 데이터 관리 시 가장 선행되는 것은?

    a. 스토리지 계정 생성

### Q16

- Azure Functions(함수 앱)에서 트리거의 의미는?

    a. 함수를 호출하는 이벤트 유형

### Q17

- 다음 중 **스토리지 다중화 및 확장 기술**이 아닌 것은?

    a. NFV

    - 스토리지 다중화 및 확장 기술
        - NAS
        - SAN
        - RAID

### Q18

- 클라우드의 서비스의 **고가용성을 확보**하기 위한 **방안**으로 올바르지 않은 것은?
    a. 중앙 집중식 서비스 구성 및 리소스 배치
    - 클라우드의 서비스의 고가용성을 확보하기 위한 방안
        - 다중 애플리케이션 서버와 부하 분산
        - 데이터베이스 혹은 스토리지 이중화
        - 리소스 풀링, 로드 밸런싱, 클라우드 버스팅 사용

### Q19

- 다음은 무엇에 대한 설명인가?
    - 엔드포인트 단말기의 물리적 위치와 **인접한 곳**에서 컴퓨팅을 수행하여 요청에 대한 **반응 시간을 단축** 시키는 기술
    a. 엣지 컴퓨팅

# 연습 문제

---

1. NAS는 서버별 별도의 파일 시스템을 제공한다.

    a. X

    - NAS는 일종의 파일 서버로 자체적으로 파일 시스템을 구축하고 네트워크 스위치나 라우터를 통해 데이터를 송수신하여 관리함
2. 로드 밸런서(L4 스위치)는 수평 스케일링 방식의 서버 처리 성능 확장 방식에 사용된다. 

    a. O

    - 수평 스케일링은 하나의 장비에서 처리하던 일을 여러 장비에 나누어 처리하는 방식이며 로드 밸런서는 처리 요청을 여러 서버로 배분하는 역할을 수행하는 네트워크 장비임
3. 클라우드, 엣지, 포그 환경에서 더 고성능의 컴퓨팅 리소스가 필요할 경우 점차적으로 클라우드에서 포그, 엣지 순으로 데이터를 전송한다. 

    a. X

    - 컴퓨팅의 성능 및 처리 범위는 엣지, 포그, 클라우드 순서로 증가함
    - 따라서 더 고성능의 리소스가 필요할 경우 엣지, 포그, 클라우드 순서로 데이터가 전송됨
