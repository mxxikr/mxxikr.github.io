IP 주소 체계와 Subnet, Supernet

IP

IP(Internet Protocol)

  • PC 간 연결하기 위한 통신 규약 (인터넷 규약)
  • 인터넷 상에 있는 컴퓨터 고유의 주소
  • 인터넷 상의 한 컴퓨터에서 다른 컴퓨터로 데이터 주고 받기 가능
  • IPv4와 IPv6로 나뉨

IPv4

  • 32 bit
  • 10진수와 2진수 사용
  • 0.0.0.0 ~ 255.255.255.255 = 2^8 * 4 = 2^32 = 약 43억개의 IP 생성 가능

IPv6

  • 128 bit
  • 16진수 사용(0~f)
  • 0.0.0.0.0.0.0.0 ~ ffff.ffff.ffff.ffff.ffff.ffff.ffff.ffff = 2^123 = 43억 * 43억 * 43억 * 43억 IP 생성 가능
  • IPv4의 고갈로 인해 생성



IP 주소 체계와 클래스

Class

  • IP의 사용에 따라 분류된 구분 값
  • Classful
    • Network ID 고정
    • 따로 알려주지않아도 어디까지가 Net ID인지를 인식
    • ex) 홍길동 02-123-4567​​
    • ex) 192.168.1.0​ → 255.255.255​​.0​​
  • Classless
    • Network ID 유동적 → 관리 효율
    • 따로 알려주지 않으면 어디까지가 Net ID인지 인식 불가능
    • Net Mask / Wildcard Mask / Prefix로 구분 가능
    • ex) 연개소문 → 연개​ 소문​/연​ 개소문​​
    • ex) 192.168.1.0 대역 255.255.255.128 → 192.168.1.0 ~ 127, 192.168.128 ~ 192.168.255​

공인 IP

  • Internet 상에서 사용하는 IP
  • 유일한 IP

사설 IP

  • ISP(Internet Service Provider) 업체를 통해서 제공 받음
  • 인터넷이 가능한 IP
  • 같은 IP 여러개 존재
  • NAT(통신망의 주소 변환기)를 통해 사설 IP를 공인 IP 주소로 변환해 인터넷에 접속
  • 단독 인터넷 사용 불가능
    • 공유기나 내부 네트워크를 사용해 인터넷에 접속할 경우 사설 IP라고 하는 특정 주소 범위(192.168.0.1~192.168.255.254)가 내부적으로 사용되고 공인 IP를 알아 낼 수 없다.
    • port forwarding이나 VPN 사용 시 사설 IP끼리 통신 가능



IP 표기법

Net Mask

  • Net ID와 Host ID를 구분해주는 구분자 (기준값)
  • 변하지 않는 부분(Net ID)를 2진수 1로 표기
  • ex) 10.0.0.0 (10.0.0.0 ~ 10.255.255.255) 대역 → 1111 1111.X.X.X → 255.0.0.0​​
  • A Class : 255.0.0.0
  • B Class : 255.255.0.0
  • C Class : 255.255.255.0

Wildcard Mask

  • NetMask를 거꾸로 주는 형태 (1111 1100 → 0000 0011)
  • 연속되지 않은 네트워크를 표현하기 위하여 오류검사를 통해 표현하는 표기법
    • 0 : 검사 시행 (바뀌지 않음)
    • 1 : 검사 미시행 (바꿀 수 있음)
  • 변하는 부분(Host ID)을 2진수 1로 표기
  • ex) 10.0.0.0 (10.0.0.0 ~ 10.255.255.255) 대역 → X.1111 1111.1111 1111.1111 1111 → 0.255.255.255​
  • A Class : 0.255.255.255
  • B Class : 0.0.255.255
  • C Class : 0.0.0.255

Prefix

  • 앞을 고정
  • Network-ID 부분의 1의 개수를 숫자로 표기
  • ex) 10.0.0.0 → 1111 1111.0.0.0 → /8

CIDR 표기법 (Classless Inter Domain Routing)

  • 네트워크 대역 표기법
  • 2진수로 표기
  • ex) 1111 1111.1111 1111.1111 1111.0000 0000 → 1의 개수 : 24개
    • Prefix : 192.168.1.0/24
    • IP : 192.168.1.1 ~ 192.168.1.254
    • Net Mask : 255.255.255.0



Subnet

Subnet

  • IP 손실 줄이고 IP 보호하기 위해 Network를 나누는 개념
  • 서로 같은 Network 영역일 때는 Switching 했지만 다른 Network 영역이 되어 Routing 하게 됨
  • 균등 Subnetting과 비균등 Subnetting으로 나눠짐
  • ex) 홍 길동 → 홍ㄱ ㅣㄹ동
  • ex) 남양 홍씨 홍 길동 → 남양군파 남양 홍씨 홍 길동 → 길자 돌림 남양군파 남양 홍씨 홍 길동

균등 Subnettnig

  • 새로 생성된 Subnet에 속하는 Host 수가 일정
  • ex) 회사 사용 network 200.1.1.0/24 → Network 1개
  • ex) 8개의 부서별로 Network를 분리(Subnet 8개 필요)해야함
    • Subnet 8개 = 2^3 = subnet-bit : 3
    • 8bit - 3bit = 5bit : host-bit = Host 32개 = 2^5

비균등 Subnetting(=VLSM)

  • 새로 생성된 Subnet에 속하는 Host수가 다름
  • 다양한 길이의 Subnet
  • 가변 길이 Subnet Mask
  • 하나의 네트워크 영역을 서로 다른 크기로 Subnetting하는 기법
  • VLSM (Variable Length Subnet Mask)라고 부름
  • ex) 회사 사용 network 201.10.1.0/24 Main Network 1개)
    • 4개의 부서별로 Network를 분리 (Subnet 4개 필요)해야 함
    • 영업부는 120개, 인사부는 60개, 관리부와 홍보부는 20개씩 Host 사용)

통신 발송 방식

BroadCast

  • NetWork 상 불특정 다수에게 통신 발송
  • IPv4에서만 사용
  • ex) 라디오, TV

Multicast

  • NetWork 상 특정 다수에게 통신 발송
  • 특정 그룹 통신
  • ex) 부산방송, 서울방송, 학교에서 1반만 수업

UniCast

  • 1 : 1 통신 발송 방식
  • ex) 통화

AnyCast

  • 가장 가까운 곳에 있는 불특정 다수 랜덤으로 통신 발송
  • 근거리 통신
  • IPv6에서만 사용
  • ex) 무전기
